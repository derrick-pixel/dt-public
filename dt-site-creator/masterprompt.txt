PART A — WORKFLOW (What To Do Before, During, and After Building)
================================================================================

A1. IMMEDIATE GITHUB REPO + FIRST PUSH
---------------------------------------
When starting any new site:
1. Create a new GitHub repository under "derrick-pixel" using `gh repo create`
2. Initialize the project, make the first commit, and push immediately
3. Enable GitHub Pages if applicable
4. Every subsequent iteration must be committed and pushed — progress is always
   live and versioned from minute one

Skip this only if Derrick explicitly says otherwise.


A2. COMPETITIVE RESEARCH (30+ SITES) — NO APPROVAL NEEDED
----------------------------------------------------------
Before writing any code for a new site, research at least 30 competitor and
reference sites in the same domain. For each competitor, analyze:

  - Layout patterns and information architecture
  - Visual design (colors, typography, imagery style)
  - Key features and interactive elements
  - Calls to action and conversion flows
  - Mobile experience and responsiveness
  - Strengths and weaknesses

After completing research, go STRAIGHT into building. Do NOT pause for Derrick's
approval — proceed directly from research to code.


A3. ADMIN PANEL — COMPETITOR ANALYTICS + MARKET INSIGHTS
---------------------------------------------------------
Every new site ships with a two-page admin panel (no password required
initially). Both pages use the same design system as the main site and are
linked together with a shared admin navigation bar.

ADMIN PAGE 1 — admin.html (Competitor Analytics Dashboard):
  - All 30+ competitors listed with their URLs
  - Per-competitor analysis: layout, design, features, CTAs, mobile UX,
    strengths, weaknesses
  - Visual comparisons: rating scales, scoring tables, radar charts where useful
  - Summary of which ideas were borrowed and which weaknesses were improved upon
  - Final design rationale: why our site beats the competition
  - Singapore & SEA landscape analysis:
    * Market size and growth trajectory
    * Key players in Singapore and Southeast Asia
    * Regional trends and emerging patterns
    * Regulatory considerations by country
    * Cultural and localization factors
    * Adoption patterns specific to the region

ADMIN PAGE 2 — admin-insights.html (Market & Pricing Insights):
  - Pricing strategies: multiple recommendations (freemium, tiered,
    pay-per-use, subscription, one-time, enterprise, etc.) with pros/cons for
    each, tailored to the specific domain
  - Target user segments: detailed personas including demographics, needs, pain
    points, behavior patterns, tech-savviness, decision-making factors
  - Willingness to pay: estimated price sensitivity per segment, benchmarks
    from competitors, suggested price points with justification
  - Next best alternatives: what users would turn to if they don't use our
    service — direct competitors, DIY solutions, manual processes, adjacent
    tools. Include switching costs and friction analysis.


A4. UNIQUE COLOR PALETTE PER SITE
-----------------------------------
Each new site MUST have a unique, creative color palette. Never reuse the exact
colors from the previous site. The accent color table (see Section B2) is a
starting guide only — go beyond it. Experiment with unexpected color
combinations, gradients, and tones that fit the project's personality. Only
reuse a previous site's palette if Derrick explicitly requests it.


A5. WHATSAPP-SHAREABLE OG THUMBNAIL ON EVERY COMMIT
-----------------------------------------------------
Every successful GitHub commit must include a working Open Graph image
(og-image.jpg, 1200x630). The OG image must reflect the site's current state —
branding, title, accent colors, tagline. If a commit changes the site's look,
content, or identity, regenerate the OG image to match BEFORE committing. The
thumbnail must always be accurate to the current version, never stale.

PART B — DESIGN SYSTEM (How Every Site Should Look)
================================================================================

B1. TECH STACK (NON-NEGOTIABLE)
--------------------------------
- Vanilla HTML5, CSS3, JavaScript — NO frameworks (no React, Vue, Next.js,
  Svelte, Angular, Tailwind)
- No build tools — no webpack, vite, or bundlers. Files are served directly.
- Google Fonts via CDN link in <head>
- External libraries only when needed: Chart.js for data viz, QRCode for
  payments — loaded via CDN. Every CDN link must justify itself.
- GitHub Pages deployment — static files only
- CSS goes in <style> in <head> (for single-page sites) or a separate .css
  file (for multi-page sites)
- JS goes in <script> at end of <body> (inline for small sites) or separate
  .js files (for complex sites)


B2. THEME — CREATIVE & UNIQUE PER SITE
----------------------------------------
No default theme. Every site gets a unique, creative visual direction — dark,
light, colorful, muted, vibrant, earthy, neon, pastel, or anything else that
fits the project. Never default to dark mode. Never repeat the same theme
approach across consecutive sites. Be bold and inventive.

All colors must be defined via CSS custom properties (:root variables):
  --bg        (deepest background)
  --surface   (card/panel background)
  --card      (elevated card background)
  --border    (subtle border)
  --border2   (stronger border)
  --text      (primary text)
  --muted     (secondary text)
  --muted2    (tertiary text)
  --accent    (primary accent)
  --accent2   (secondary accent)

B7. ANIMATIONS
---------------

SCROLL REVEAL (IntersectionObserver):
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(e => {
      if (e.isIntersecting) e.target.classList.add('visible');
    });
  }, { threshold: 0.12 });
  document.querySelectorAll('.animate').forEach(el => observer.observe(el));

  .animate { opacity: 0; transform: translateY(24px);
    transition: opacity 0.6s ease, transform 0.6s ease; }
  .animate.visible { opacity: 1; transform: translateY(0); }

COUNTER ANIMATION:
  function animateCounter(el, target, suffix = '') {
    const duration = 1800;
    const start = performance.now();
    function tick(now) {
      const progress = Math.min((now - start) / duration, 1);
      const eased = 1 - Math.pow(1 - progress, 3);
      el.textContent = Math.round(target * eased) + suffix;
      if (progress < 1) requestAnimationFrame(tick);
    }
    requestAnimationFrame(tick);
  }

HOVER TRANSITIONS (all interactive elements use transition: all 0.2s ease):
  Cards:   translateY(-4px) + shadow increase
  Buttons: translateY(-1px) + brightness or glow
  Links:   color change to accent
  Images:  scale(1.03) + slight brightness


B8. JAVASCRIPT PATTERNS
-------------------------
Organization:
  - Utility functions at top of script
  - Event listeners grouped by feature
  - DOM references via getElementById() and querySelectorAll()
  - No jQuery, no lodash — pure vanilla JS
  - State stored in localStorage when persistence is needed
  - Inline onclick for simple actions, addEventListener for complex logic

Mandatory utility functions every site includes:

  // XSS prevention
  function esc(s) {
    return (s||'').replace(/&/g,'&amp;').replace(/</g,'&lt;')
      .replace(/>/g,'&gt;').replace(/"/g,'&quot;');
  }

  // Toggle mobile menu
  function toggleMenu() {
    document.getElementById('mobile-menu').classList.toggle('open');
    document.getElementById('hamburger').classList.toggle('open');
  }

  // Show toast notification
  function showToast(msg, duration = 2200) {
    const t = document.getElementById('toast');
    t.textContent = msg;
    t.classList.add('show');
    setTimeout(() => t.classList.remove('show'), duration);
  }

  // Smooth scroll to section
  document.querySelectorAll('a[href^="#"]').forEach(a => {
    a.addEventListener('click', e => {
      e.preventDefault();
      document.querySelector(a.getAttribute('href'))
        ?.scrollIntoView({ behavior: 'smooth' });
    });
  });

PART C — DESIGN PRINCIPLES
================================================================================

1.  SHIP FAST — No build steps, no npm install. HTML opens in browser instantly.
2.  BOLD AND CREATIVE — Every site gets a unique visual identity. High contrast.
    Striking color choices. No repeated themes across consecutive sites.

5.  MOBILE-FIRST RESPONSIVE — Works on phone. Hamburger menu. Stacked layouts.
6.  SOCIAL-READY — Full Open Graph tags on every page. OG image on every commit.
7.  SELF-CONTAINED — Each project is a standalone folder. No shared deps.
8.  GLASSMORPHIC NAV — Fixed, blurred, semi-transparent navigation bar.
9.  GRADIENT ACCENTS — linear-gradient(135deg, accent1, accent2) for hero text,
    progress bars, CTAs.
10. GENEROUS WHITESPACE — 80px section padding, 24px card padding, 28px gaps.
