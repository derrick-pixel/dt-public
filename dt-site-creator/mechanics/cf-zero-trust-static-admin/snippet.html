Cloudflare dashboard config
  1. Account/Zone setup: Ensure your domain is proxied through Cloudflare DNS (orange-cloud). CF Pages deployments are automatically proxied.
  2. Enable Zero Trust: In the Cloudflare dashboard, go to Zero Trust → Access → Applications → Add an Application.
  3. Application type: Select Self-hosted.
  4. Application URL pattern: yourdomain.com/admin/* (or /intel/*, /aevum/admin/*, etc.)
  5. Identity provider — Google OAuth (do this first):
    • Go to Zero Trust → Settings → Authentication → Login methods → Add new.
    • Select Google.
    • In Google Cloud Console, create an OAuth 2.0 Client ID (Web application type).
    • Add authorised redirect URI: https://<your-team>.cloudflareaccess.com/cdn-cgi/access/callback
    • Paste the Client ID and Client Secret into the CF form.
    • Save. Do NOT enable OTP as a fallback if SG email delivery is a concern (see pitfall bba-cf-access-otp-unreliable).
  6. Policy:
    • Action: Allow
    • Include rule: Emails ending in @yourdomain.com — or list specific email addresses.
  7. Session duration: 24 hours is a reasonable default for internal admin tooling. Reduce to 4 hours for sensitive paths.
  8. Test in incognito: You should be redirected to Google, then back to your admin page. The CF_Authorization cookie will be set by Cloudflare's edge.
Sample policy JSON export

CF Access does not natively export policies as JSON files, but you can version-control your policy shape for documentation or Terraform/API migration purposes. The shape below matches the CF Access API. 

{
  "name": "Allow admin emails",
  "decision": "allow",
  "include": [
    { "email_domain": { "domain": "yourdomain.com" } }
  ],
  "require": [],
  "exclude": []
}

To use the CF API directly: PUT /accounts/<account_id>/access/apps/<app_id>/policies/<policy_id>

For a specific-emails variant: 

{
  "name": "Named admin list",
  "decision": "allow",
  "include": [
    { "email": { "email": "admin@yourdomain.com" } },
    { "email": { "email": "partner@external.com" } }
  ],
  "require": [],
  "exclude": []
}
Sign-out

Cloudflare Access provides a built-in logout endpoint that clears the CF_Authorization cookie and invalidates the session.

Sign out of this admin session

You can also add a sign-out button in your admin nav: 

<a href="/cdn-cgi/access/logout">Sign out</a>
