# Dashboard-Analytics — Mechanic Fit

| Mechanic | Fit | Notes |
|---|---|---|
| og-social-meta | core | Mandatory |
| chartjs-dashboard | core | Data visualization backbone |
| admin-auth-gate | core | Dashboards must be auth-gated |
| localstorage-state | core | Filter / view preferences |
| admin-insights-panel | core | Document WHY the KPIs/thresholds were chosen |
| dashboard-admin-shell | core | For internal CRMs / ops tools — full admin app scaffold |
| streamlit-analytics-scaffold | core | When HTML + Chart.js can't handle the math |
| multi-page-scaffold | optional | If multiple dashboards in one app |
| wizard-form | optional | Report generators, filter builders |
| pdf-pipeline | optional | Export reports |
| design-variant-switcher | rare | Dashboards usually have one aesthetic |
| formspree-form | rare | Dashboards usually have auth'd in-app flows, not email capture |
| canvas-hero | rare | Usually out-of-place |
| paynow-qr | rare | Not transactional |
