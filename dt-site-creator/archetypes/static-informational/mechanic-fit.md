# Static Informational — Mechanic Fit

Human-readable summary. Authoritative source: each mechanic's `meta.json.fits["static-informational"]`.

| Mechanic | Fit | Notes |
|---|---|---|
| og-social-meta | core | Mandatory on every site |
| multi-page-scaffold | core | Standard for content-heavy sites |
| formspree-form | core | Lead capture / contact form for sites without a backend |
| admin-insights-panel | optional | For strategy-heavy marketing sites (pairs with /admin.html competitor dashboard) |
| design-variant-switcher | optional | Great when brand direction is unsettled — client picks a winner |
| canvas-hero | optional | Use for tech/defence/aviation domains |
| wizard-form | optional | For lead-capture diagnostic or ROI calculator |
| admin-auth-gate | optional | Add for competitor-analytics admin panels |
| localstorage-state | rare | Only if site has quiz / progress / preferences |
| chartjs-dashboard | rare | Only for data-visualization panels within marketing pages |
| pdf-pipeline | rare | Only if site outputs a PDF (e.g., proposal generator) |
| dashboard-admin-shell | rare | Static sites rarely need a full admin app |
| streamlit-analytics-scaffold | rare | Use dashboard-analytics archetype instead |
| paynow-qr | rare | Use transactional archetype instead |
